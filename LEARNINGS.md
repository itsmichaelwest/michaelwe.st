# Learnings

- 2026-02-21: motion@12 uses `motion/react` import path (not `framer-motion`). `layoutId`, `AnimatePresence`, `LayoutGroup` all from `motion/react`.
- 2026-03-21: Gallery full-screen view uses GalleryDetail overlay (native scroll, crisp text) + GalleryItem (image transform animation only). The overlay crossfades with GalleryItem's image during open/close transitions. Touch handlers for swipe/dismiss live on GalleryDetail's image wrapper, not GalleryItem.
- 2026-03-21: `npm run build` has a pre-existing robots.txt PageNotFoundError — unrelated to gallery code, likely Next.js 15/16 compatibility issue.
- 2026-04-19: Next.js 16 renamed `middleware.ts` → `proxy.ts` and the exported `middleware` function → `proxy`. Old names still work but emit deprecation warning.
- 2026-04-19: Markdown content negotiation implemented via `proxy.ts` + `/api/md/*` route handlers (force-static + generateStaticParams). Proxy inspects `Accept` header, rewrites to markdown route if `text/markdown` q >= `text/html` q. Response headers on markdown: `content-type: text/markdown; charset=utf-8`, `x-markdown-tokens` (chars/4 estimate), `vary: Accept`.
- 2026-04-19: `robots.txt` is generated by `app/robots.txt/route.ts` (route handler, force-static) instead of `app/robots.ts` — needed to emit a custom `Content-Signal:` line which `MetadataRoute.Robots` doesn't support. Don't reintroduce `app/robots.ts` or it will collide.
- 2026-04-19: `next.config.js` `headers()` supports `:param` substitution in header VALUES (not just `source`), e.g. `/work/:id` can emit `Link: </api/md/work/:id>`. Header `Vary` set via `headers()` is overwritten by Next.js 16's own framework `Vary: rsc, next-router-state-tree, ...` on RSC routes — don't rely on it.
- 2026-04-19: Killing `next start` / `next dev` on Windows: `taskkill /F /IM node.exe` can miss the pnpm-spawned node. Use `Get-NetTCPConnection -LocalPort 3000 | Stop-Process -Force` via powershell to reliably free the port.
